# STATUS

## 概览

`quanttide-laboratory-of-data-engineering` 是量潮数据工程的**实验田**——用于验证想法、跑通原型、积累经验，成熟后再沉淀到正式模块中。

## 当前状态：v0.1.0-alpha.1

### 已实现

- **项目骨架**：`quanttide_data_agents` 包结构，pdm 管理依赖，Python 3.8+
- **数据标注实验**：`examples/data_annotation.py`
  - 调用通义千问（`qwen-max-longcontext`）对裁判文书进行结构化信息提取
  - 输出案件类型、当事人、时间、地点、经济损失等 25 个字段的 JSON
  - 已验证一个样本（交通肇事二审案件），`actual_result.json` 与 `excepted_result.json` 基本吻合

### 依赖

- `dashscope` — 阿里云通义千问 API
- `python-dotenv` — 环境变量管理

### 文档

- `docs/README.md` — 用户文档（目前为空标题）
- `CHANGELOG.md` — 仅记录 v0.1.0 初始版本

## 差异（actual vs excepted）

当前标注结果与期望结果在以下字段存在细微差异，反映了大模型在处理细节上的不稳定：

| 字段 | excepted | actual |
|---|---|---|
| 文化程度 | `"初中文化"` | `"初中"` |
| 案发时间 | `"2018年10月28日"` | `"2020年1月13日18时许"` |
| 实习期类型 | `"无"` | `"C1"` |
| 经济损失 | `"760956.5"` | `"760956.5元"` |
| 撤销案件号 | `"无"` | `"（2020）陕0825刑初53号"` |

> 需人工判断哪些是模型精度问题、哪些是期望标注本身的偏差。
